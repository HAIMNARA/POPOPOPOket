# 통합 함정 아카이브 (Unified Pitfalls Archive)

포켓몬스터 에이전트 하네스(Harness) 시스템을 개발하고 운영하는 과정에서 마주친 다양한 기술적 함정(Pitfalls)들을 정리한 문서입니다. 이 아카이브는 환경 설정, 비전/입력, 진행도/메트릭, 시스템 설계, 운영, 그리고 강화학습(RL) 분야에서 발생한 시행착오들을 기록하여 향후 유사한 문제의 재발을 방지하는 데 목적이 있습니다.

---

## 1. 환경 설정 함정 (Environment Configuration Pitfalls)

### 1. mGBA --script CLI 옵션 미존재 (POPOPOPOket #1)
- **현상**: mGBA 에뮬레이터를 명령줄(CLI)에서 실행할 때 `--script` 옵션을 지정했으나, 에뮬레이터가 스크립트를 로드하지 못하고 즉시 종료되는 문제가 발생했습니다.
- **원인**: 사용 중인 mGBA 빌드 버전이 Lua 스크립팅을 지원하지 않거나, CLI 옵션 파싱 규칙이 변경되었기 때문입니다.
- **해결책**: Lua 스크립트 지원이 명시된 mGBA 버전을 사용하고, 스크립트 경로를 절대 경로로 전달해야 합니다.

### 2. Lua 소켓 서버 2회 로드 / dofile() 크래시 (POPOPOPOket #2)
- **현상**: 에뮬레이터 내에서 Lua 스크립트를 실행할 때 소켓 서버가 중복 로드되거나 `dofile()` 함수가 반복 호출되면서 에뮬레이터가 크래시되었습니다.
- **원인**: 이전 세션의 소켓 포트가 닫히지 않은 상태에서 새 스크립트가 동일한 포트로 바인딩을 시도했기 때문입니다.
- **해결책**: 스크립트 시작 부분에 기존 소켓 연결을 명시적으로 닫는 예외 처리 코드를 추가하고, 중복 로드를 방지하는 가드를 세웠습니다.

### 3. Lua GUI 자동화 불안정 (POPOPOPOket #3)
- **현상**: 윈도우 환경에서 한글이 포함된 경로를 사용하거나 다중 윈도우가 띄워진 상태에서 Lua GUI API가 오작동했습니다.
- **원인**: Lua 엔진의 문자열 인코딩 한계와 창 포커스 유실로 인한 현상입니다.
- **해결책**: 프로젝트 경로에서 한글을 배제하고 영문 경로만 사용하도록 강제했으며, 창 포커스에 의존하지 않는 백그라운드 API를 사용하도록 수정했습니다.

### 4. 에이전트에게 "게임 실행" 위임 시 무한루프 사고 (POPOPOPOket #15)
- **현상**: LLM 에이전트에게 에뮬레이터 프로세스 실행 권한을 부여하자, 에뮬레이터가 켜지지 않을 때마다 프로세스를 새로 띄워 수십 개의 에뮬레이터가 실행되는 무한루프 사고가 발생했습니다.
- **원인**: 프로세스 실행 여부를 확인하는 상태 체크 로직이 미흡했습니다.
- **해결책**: 에뮬레이터 실행은 외부 런처 스크립트가 전담하도록 격리하고, 에이전트는 이미 실행된 포트에 연결만 하도록 제한했습니다.

## 2. 비전 및 입력 함정 (Vision & Input Pitfalls)

### 5. 이미지 오버레이 오염으로 인한 모델 추론 저하 (POPOPOPOket #14)
- **현상**: 게임 화면 위에 좌표 그리드와 텍스트를 직접 겹쳐서 렌더링하자, 모델이 화면 속의 포켓몬이나 대화창 글자를 인식하지 못하는 현상이 발생했습니다.
- **원인**: 오버레이된 선과 글자가 원본 이미지의 픽셀 정보를 오염시켰기 때문입니다.
- **해결책**: 원본 이미지와 가이드 이미지를 분리하여 멀티모달 입력으로 각각 제공하는 구조로 변경했습니다.

### 6. 풀숲 오인식으로 인한 이동 차단 문제 (POPOPOPOket #10)
- **현상**: 맵 마스킹 시스템이 풀숲 타일을 벽(red)으로 분류하여, 에이전트가 풀숲을 지나가지 못하고 먼 길을 우회하거나 갇히는 문제가 발생했습니다.
- **원인**: 풀숲을 안전한 타일(green)로 분류하면 배틀 확률을 인지하지 못하고, 벽(red)으로 분류하면 이동이 불가능하다고 판단하는 이분법적 설계의 한계였습니다.
- **해결책**: 풀숲을 위한 제3의 카테고리인 노란색(yellow) 마스킹을 도입하여 해결했습니다.

### 7. 침실 스폰 계단 오인식 문제 (POPOPOPOket #11)
- **현상**: 게임 시작 직후 주인공의 침실에서 아래층으로 내려가는 계단을 에이전트가 인식하지 못하고 방 안을 맴돌았습니다.
- **원인**: 계단 타일이 화면 가장자리에 위치하여 비전 모델이 이를 벽으로 오인했기 때문입니다.
- **해결책**: 침실 스폰 시점에 한해 "아래층으로 내려가기 위해 계단으로 이동하라"는 명시적인 가이드 프롬프트를 주입했습니다.

## 3. 진행도 및 메트릭 함정 (Progress & Metrics Pitfalls)

### 8. Viridian mapId=0x01 충돌 (POPOPOPOket #5)
- **현상**: 상록시티(Viridian City)의 맵 ID가 `0x01`로 지정되어 있어, 첫 번째 맵 전환 테스트 코드와 충돌을 일으켰습니다.
- **원인**: 테스트 코드가 맵 ID `0x01`을 태초마을(Pallet Town) 내부의 특정 건물로 하드코딩하여 가정했기 때문입니다.
- **해결책**: 테스트 코드의 맵 ID 검증 로직을 유연하게 수정하고 하드코딩을 제거했습니다.

### 9. ROM 정체성 혼동 (POPOPOPOket #8)
- **현상**: 포켓몬스터 적(Red) 버전을 실행 중임에도 메트릭 수집 모듈이 금(Gold) 버전의 메모리 주소를 참조하여 엉뚱한 데이터를 수집했습니다.
- **원인**: 다중 ROM 지원을 위해 작성된 설정 파일이 잘못 로드되었습니다.
- **해결책**: ROM 파일의 해시값을 검사하여 게임 버전을 자동으로 식별하고 해당하는 메모리 맵을 바인딩하도록 수정했습니다.

### 10. 토큰 절감만을 추구하다 진행도가 악화되는 롤백 대상 사례 (POPOPOPOket #8)
- **현상**: LLM API 비용을 줄이기 위해 프롬프트 크기를 대폭 축소했으나, 에이전트의 상황 판단 능력이 떨어져 게임 진행 속도가 느려지고 같은 자리를 맴돌았습니다.
- **원인**: 필요한 컨텍스트 정보까지 삭제하여 발생한 성능 저하입니다.
- **해결책**: 토큰 소모량 감소가 게임 진행도 메트릭을 악화시킨다면 즉시 이전 프롬프트 템플릿으로 롤백하는 원칙을 수립했습니다.

### 11. 증거 없는 개선 금지 원칙
- **현상**: 개발자의 직관에 의존하여 프롬프트나 코드를 수정했으나, 실제 벤치마크 테스트에서 성능이 저하되는 일이 빈번했습니다.
- **원인**: 정량적인 데이터 검증 없이 감행한 수정 때문입니다.
- **해결책**: 모든 변경 사항은 최소 3회 이상의 에피소드 실행 데이터와 로그 분석을 통한 정량적 증거가 있을 때만 메인 브랜치에 반영하도록 제한했습니다.

## 4. 시스템 설계 함정 (System Design Pitfalls)

### 12. 대화/메뉴 RAM 감지 불안정 시 visual-fallback 부재 (POPOPOPOket #6)
- **현상**: 특정 대화창이나 메뉴가 열렸을 때 RAM 주소의 값이 즉각적으로 변하지 않아 시스템이 대화 상태를 감지하지 못하고 멈췄습니다.
- **원인**: 게임 엔진의 내부 상태 업데이트 지연 때문입니다.
- **해결책**: RAM 감지가 실패하더라도 화면 캡처 이미지 분석을 통해 대화창의 존재를 감지하는 시각적 예외 처리(visual-fallback) 메커니즘을 추가했습니다.

### 13. fast-flow 바이패스 위치 오류 (POPOPOPOket #7)
- **현상**: 대화창을 빠르게 넘기기 위한 fast-flow 바이패스 로직이 제대로 작동하지 않고 대화가 지연되었습니다.
- **원인**: 해당 로직이 상위 감독 모듈(Supervisor)에 위치하여, 실제 매 턴 입력을 처리하는 `index.ts`의 메인 턴 루프를 우회하지 못했습니다.
- **해결책**: fast-flow 바이패스 로직을 `index.ts` 내부의 최우선 실행 루프로 이전하여 대화창 감지 시 즉각적인 A/B 탭이 입력되도록 수정했습니다.

### 14. 단일 에뮬레이터 원칙 위반으로 인한 포트 충돌 (POPOPOPOket #9)
- **현상**: 병렬 테스트 실행 시 여러 에뮬레이터 인스턴스가 동일한 포트(예: 8888)로 바인딩을 시도하여 네트워크 에러가 발생했습니다.
- **원인**: 인스턴스별 포트 할당 로직이 부재했습니다.
- **해결책**: 각 에뮬레이터 인스턴스가 고유한 포트를 동적으로 할당받아 실행되도록 포트 관리자를 도입했습니다.

## 5. 운영 함정 (Operational Pitfalls)

### 15. Lazy Codex 콜 호스트 드롭으로 인한 병렬 구조 중단 (POPOPOPOket #12)
- **현상**: 대규모 병렬 테스트 중 특정 호스트가 응답을 멈추면서 전체 테스트 파이프라인이 대기 상태에 빠졌습니다.
- **원인**: 호스트 연결에 대한 타임아웃 설정이 없어서 무한 대기가 발생했습니다.
- **해결책**: 모든 외부 API 및 호스트 호출에 최대 30초의 타임아웃을 설정하고, 실패 시 해당 호스트를 제외하고 진행하도록 수정했습니다.

### 16. 다중 에뮬레이터 게이트웨이 연결 오류 (POPOPOPOket #13)
- **현상**: 게이트웨이 서버가 여러 에뮬레이터의 상태를 수집하는 과정에서 데이터가 섞이거나 엉뚱한 인스턴스로 명령이 전달되었습니다.
- **원인**: 세션 ID와 소켓 매핑이 올바르게 관리되지 않았습니다.
- **해결책**: 모든 요청 헤더에 고유한 `client-id`를 필수적으로 포함하도록 규정하고 라우팅 테이블을 강화했습니다.

### 17. 토크나이저/모델 토큰 소비 측정 함정 (POPOPOPOket #4)
- **현상**: 로컬 토크나이저로 계산한 토큰 수와 실제 LLM API 제공업체가 청구한 토큰 수 사이에 큰 오차가 발생했습니다.
- **원인**: 이미지 입력(비전 토큰)의 계산 방식이 로컬 토크나이저에 반영되지 않았기 때문입니다.
- **해결책**: 이미지 크기에 따른 API 스펙상의 비전 토큰 계산 공식을 반영하여 측정 로직을 보정했습니다.

## 6. RL 특유 함정 (PokemonRedExperiments)

### 18. 보상 해킹 (Reward Hacking)
- **현상**: 에이전트가 스토리를 진행하지 않고, 특정 구역에서 포켓몬 센터를 무한히 드나들거나 메뉴를 열고 닫는 행동만 반복하며 보상을 챙겼습니다.
- **원인**: 특정 상태 변화에 부여한 보상 값이 너무 커서 발생한 부작용입니다.
- **해결책**: 보상 함수를 설계할 때 상태 변화의 1회성 획득에만 보상을 주고, 반복 행동에는 패널티를 부여하도록 보상 구조를 개편했습니다.

### 19. KNN 탐색 오버헤드 문제
- **현상**: 에이전트의 위치 다양성을 측정하기 위해 KNN(K-Nearest Neighbors) 알고리즘을 사용했으나, 에피소드가 길어질수록 탐색 연산 시간이 기하급수적으로 증가했습니다.
- **원인**: 메모리에 누적되는 좌표 데이터의 양이 너무 많아졌기 때문입니다.
- **해결책**: KNN 방식 대신 맵의 격자 좌표 방문 횟수를 직접 카운트하는 V2 좌표 테이블 방식으로 전환하여 연산 오버헤드를 제거했습니다.

### 20. 로컬 최적 함정 (Local Minima)
- **현상**: 에이전트가 특정 동굴이나 미로에서 빠져나오지 못하고 동일한 구역만 맴돌며 학습이 정체되었습니다.
- **원인**: 탐색(Exploration)보다 활용(Exploitation)에 치우쳐 새로운 경로를 시도하지 않았기 때문입니다.
- **해결책**: 에피소드 초기화 시 무작위 위치에서 시작하는 기법과 탐색 확률(Epsilon) 스케줄러를 도입하여 다양성을 보존했습니다.

## 7. 빠른 진단 체크리스트 (Quick Diagnosis Checklist)

| 증상 | 1차 확인 사항 | 해결 방향 |
| :--- | :--- | :--- |
| mGBA 시작 즉시 종료 | CLI 옵션에 `--script`가 포함되어 있는가? | Lua 지원 빌드 확인 및 절대 경로 사용 |
| `:5000` 포트 미응답 | 에뮬레이터가 `:8888` 포트에서 정상 리스닝 중인가? | 포트 바인딩 상태 및 방화벽 확인 |
| mGBA 크래시 발생 | Lua 스크립트가 중복 로드(`dofile`)되었는가? | 소켓 해제 코드 및 중복 실행 방지 가드 확인 |
