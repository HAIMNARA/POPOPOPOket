# 09 · 향후 로드맵

우선순위는 **P0**(즉시) > **P1**(다음 이벤트) > **P2**(장기)로 구분.

## P0 — 현 하네스 즉시 개선

### 컨텍스트 압축 & 세션 핸드오프

| 문제 | 해결 방향 |
|------|-----------|
| 장시간 실행 시 컨텍스트 메모리 포화 | **컴팩션(compaction)**: 이력을 MD 파일로 정규화 |
| 새 세션 전환 시 지식 손실 | **핸드오프 프로토콜**: 진행 상태 + 실패 기록 + 현재 위치를 구조화된 형식으로 전달 |
| 불필요한 이력 누적 | **프루닝(pruning)**: 반복 실패 기록, 오래된 관찰 등을 정리 |

### 풀숲 오인식 해결

- 에이전트가 통과 가능한 풀숲을 벽으로 인식하는 문제
- **해결 방안**: 메모리 맵 마스킹에 풀숲 타일을 별도 색상(노란색 등)으로 분류
  - 초록 = 이동 가능 빈 타일
  - 노란 = 이동 가능 풀숲 타일 (전투 발생 가능)
  - 빨강 = 이동 불가 (벽/장애물)

### 초기 스폰 가이드

- 침실(2F) 스폰 시 계단이 화면 밖에 있어 에이전트가 탈출 못하는 문제
- **해결 방안**: 침실 맵 진입 감지 시 "위(UP)로 이동하여 계단을 찾아라" 가이드 프롬프트 주입

## P1 — 다음 이벤트 목표

### pyboy 프레임워크 도입

```
현재: mGBA + mGBA-http + Lua 소켓 → TypeScript 하네스
향후: pyboy (Python) → 에뮬레이터 연동 스크립팅 부담 대폭 감소
```

- **pyboy**: Python 기반 Game Boy 에뮬레이터 라이브러리
- mGBA + Lua + HTTP 브리지 체인을 **단일 Python 패키지**로 대체
- 스크립팅 연동 삽질(Lua GUI 로드, 소켓 중복 크래시 등)을 근본적으로 해결
- 하네스 설계/아키텍처에 집중할 수 있는 환경 확보

### 다층 아키텍처 표준화

회의에서 드러난 3가지 아키텍처 패턴을 표준 모듈로:

| 패턴 | 핵심 | 적용 시나리오 |
|------|------|---------------|
| 신경계 모사 (4-A) | L1 무의식 + L2 크리틱 + L3 정책 LLM | 단일 인스턴스, 범용 도메인 |
| 병렬 탐색 (4-B) | 서브 에이전트 병렬 + 상위 리즈닝 종합 | 분기가 많은 구간 |
| 중앙 공략집 (4-C) | 다중 인스턴스 + 공략집 자동 업데이트 | 장기 실행, 자가 개선 |

### 포켓몬 포획 이벤트

- 단순 이동 클리어를 넘어 **포켓몬 포획**을 목표로 하는 별도 이벤트 개최 제안
- 전투 → 약화 → 몬스터볼 투척 → 포획 판정의 복합 의사결정 필요
- 현재 어떤 참가자도 포획 성공 사례 없음 → 도전적 목표

### MCP 서버 활용 다중 에뮬레이터

- **MCP(Model Context Protocol) 서버**를 활용하면 여러 에뮬레이터 인스턴스를 동시 운용 가능
- 병렬 하네스 인스턴스 + 중앙 공략집 구조의 기반 인프라

## P2 — 장기 목표

### 완전 자동화 에이전트

> 코딩 에이전트가 하네스 개선을 자동으로 판단·수행하는 완전 자동화.

현재 상태:
- 인간(휴먼 에이전트)이 게임 관찰 → 시스템 보완 → 개선 반복
- **바이오 토큰** 기반 → 인간 인지 자원 소모

목표 상태:
- 코딩 에이전트가 게임 에이전트의 행동 관찰 → 하네스 코드 자동 수정 → 재실행
- **API 토큰** 기반 → 24시간 무중단 개선 루프

### Local Optima 견제 메커니즘

- 연합 강화 학습 프레임워크에서 알려진 **지역 최적값 함정** 방지
- 상위 웹 서버 에이전트가 전체 형세를 모니터링
- 다양성 보존: 탐색(exploration) vs 활용(exploitation) 균형

### 장기 메모리 시스템

- 세션 간 지속되는 장기 메모리
- 게임 전체에 걸친 전략적 지식 축적
- 실패/성공 패턴의 영구 아카이브

## 기술 스택 진화 방향

```
현재 (v1):
  mGBA + Lua + mGBA-http → TypeScript 하네스
  단일 인스턴스 + LLM 매 턴 호출

다음 (v2):
  pyboy (Python) → Python/TS 하이브리드 하네스
  다층 아키텍처 + 메모리 맵 마스킹 + 세션 핸드오프

장기 (v3):
  pyboy + MCP 서버 → 다중 인스턴스
  중앙 공략집 자동 업데이트 + 코딩 에이전트 자동 개선
  완전 자동화 피드백 루프
```
