# 막힘 감지 및 복구 메커니즘 (Stuck Detection & Recovery)

포켓몬스터 게임을 플레이하는 자율 에이전트는 복잡한 지형, NPC의 무작위 이동, 대화창 루프 등으로 인해 특정 구역에 갇히거나 무한 루프에 빠질 위험이 큽니다. 이러한 상황을 신속하게 감지하고 탈출하기 위해 다양한 막힘 감지(Stuck Detection) 및 복구(Recovery) 메커니즘이 설계되었습니다. 본 문서에서는 Grokemon, POPOPOPOket, 그리고 PokemonRedExperiments 프로젝트의 막힘 감지 및 복구 시스템을 상세히 다룹니다.

---

## 1. Grokemon StuckDetector

Grokemon 프로젝트의 `StuckDetector`는 약 432줄의 코드로 구현되어 있으며, 에이전트의 행동 패턴과 위치 변화를 다각도로 분석하여 막힘 상태를 감지합니다. 이 시스템은 5가지 수준의 감지 알고리즘을 실행합니다.

### 5레벨 감지 알고리즘 (5-Level Detection)

1. **actionLoop (액션 루프)**
   - 에이전트가 동일한 액션을 4회 이상 반복하고, 그동안 캐릭터의 위치가 5회 이상 변하지 않고 안정적으로 유지되는 상태를 감지합니다. 벽을 향해 계속 걷거나 닫힌 문을 계속 열려고 시도하는 상황이 이에 해당합니다.

2. **actionCycle (액션 사이클)**
   - 에이전트가 단순 반복이 아닌, 특정 패턴의 액션 시퀀스를 순환하며 반복하는 상태를 감지합니다. 예컨대 `[Up, Right, Down, Left]`와 같은 입력을 계속해서 순환 입력하는 경우입니다.

3. **locationLoop (위치 루프)**
   - 에이전트가 방문한 위치 좌표들의 이력을 추적하여, 최근 방문한 위치들이 특정 주기 내에서 계속 반복되는 상태를 감지합니다. 두 개 혹은 세 개의 타일 사이를 무한히 왕복하는 상황을 잡아냅니다.

4. **noProgress (전진 없음)**
   - 일정 시간 동안 새로운 타일을 탐색하지 못하거나 게임의 전반적인 진행도(스토리 플래그, 맵 전환 등)에 변화가 없는 상태를 감지합니다.

5. **backtrackLoop (역추적 루프)**
   - 캐릭터의 위치가 앞으로 나아갔다가 다시 뒤로 돌아오는 진동(왔다갔다) 현상을 반복하는 상태를 감지합니다. 좁은 통로에서 장애물이나 NPC에 막혀 전진과 후퇴를 반복할 때 주로 발생합니다.

### 심각도 분류 (Severity Levels)

감지된 막힘 상태는 심각도에 따라 세 가지 단계로 분류되어 후속 복구 절차로 연계됩니다.

- **none**: 정상 상태이며 특별한 개입이 필요하지 않습니다.
- **soft**: 가벼운 막힘 상태입니다. 입력 지연을 주거나 무작위 입력을 1~2회 주입하여 자연스러운 탈출을 유도합니다.
- **hard**: 심각한 막힘 상태입니다. 에뮬레이터 상태를 이전 세이브 포인트로 롤백하거나, 강제 개입 루프를 실행하여 에이전트의 제어권을 일시적으로 회수합니다.

## 2. POPOPOPOket stuck-memory

POPOPOPOket 프로젝트는 메모리 기반의 막힘 감지 및 프롬프트 주입 시스템을 사용하여 LLM(Large Language Model) 에이전트가 스스로 막힘 상태를 인지하고 대안을 찾도록 돕습니다.

### 이동 시도 추적 (Movement Tracking)

- 시스템은 에이전트가 입력한 명령 중 단일 방향 버튼(Up, Down, Left, Right)만을 실제 이동 시도로 간주하여 추적합니다. 메뉴 열기나 대화 넘기기 등의 입력은 이동 시도 카운트에서 제외합니다.

### 정지 확인 (isStationary) 및 상태 기반 키

캐릭터가 실제로 움직이지 못하고 정지해 있는지 판단하기 위해 상태 기반 키를 생성합니다.

- **RAM 데이터가 유효한 경우**: `map:<id>:x:<x>:y:<y>:dir:<dir>` 형식의 키를 생성하여 이전 턴과 비교합니다. 맵 ID, X/Y 좌표, 바라보는 방향이 모두 동일하면 정지 상태로 판정합니다.
- **RAM 데이터를 읽을 수 없는 경우**: 화면 캡처 이미지의 SHA-256 해시값 중 첫 12자를 키로 사용합니다. 화면 변화가 전혀 없다면 정지 상태로 판정합니다.

### 실패 엣지 누적 (Failed Edge Accumulation)

- 특정 컨텍스트에서 특정 액션을 수행했으나 이동에 실패한 경우, `<context>|<action>` 형태의 키를 생성하고 실패 횟수를 누적합니다.
- **STUCK_THRESHOLD**: 동일한 실패 키의 누적 횟수가 8회에 도달하면 `stuckEvents` 카운터를 증가시키고 막힘 상태로 정의합니다.
- **MAX_FAILED_EDGES**: 메모리 오버헤드를 방지하기 위해 최대 6개의 실패 엣지만 유지하며, 새로운 실패가 감지되면 가장 오래된 실패 기록을 드롭합니다.

### 복구 시도 기록 (Recovery Attempts)

- 막힘 상태를 해결하기 위해 실행한 복구 행동의 이력을 최대 6개(`MAX_RECOVERY_ATTEMPTS=6`)까지 기록합니다.

### 프롬프트 주입 형식 (Prompt Injection Format)

감지된 막힘 정보와 복구 이력은 LLM의 컨텍스트 프롬프트에 다음과 같은 정형화된 텍스트로 주입됩니다. 이를 통해 에이전트는 자신이 어디서 실패했는지 학습하고 다른 행동을 선택합니다.

```
failed movement memory:
- map=12 x=5 y=8 facing=up; hold:Up; failed 8x; last turn 42
recent recovery attempts:
- turn 43: tap:{"button":"B"} after map=12 x=5 y=8 facing=up
```

## 3. PokemonRedExperiments stuck penalty

강화학습 기반의 PokemonRedExperiments에서는 에이전트가 특정 구역을 무한히 맴도는 로컬 미니마(Local Minima)에 빠지는 것을 방지하기 위해 보상 함수에 패널티를 부여합니다.

- **좌표 방문 횟수 초과 패널티**: 에이전트가 전체 에피소드 동안 특정 좌표를 방문한 횟수가 600회를 초과하는 경우, 해당 좌표를 방문할 때마다 `-0.05` 점의 보상 패널티를 부여합니다. 이 패널티는 에이전트가 이미 탐색한 지역에 안주하지 않고 새로운 지역으로 나아가도록 유도하는 강력한 동기가 됩니다.

## 4. Grokemon InterventionLoop

`InterventionLoop`는 약 123줄의 코드로 작성된 강제 복구 모듈입니다. 에이전트가 심각한 막힘 상태(`hard stuck`)에 빠졌을 때 작동합니다.

- **동작 방식**: 에이전트의 입력을 차단하고, 미리 정의된 최대 20개의 안전한 입력 시퀀스를 에뮬레이터에 직접 주입합니다.
- **유효 버튼**: A, B, Up, Down, Left, Right, Start, Select 버튼을 조합하여 사용합니다.
- **프레임 제어**: 각 입력은 1프레임에서 60프레임 사이의 정밀한 타이밍으로 실행됩니다.
- **중단 조건**: 주입 과정 중 에뮬레이터가 입력을 거부하거나 비정상적인 상태 변화를 감지하면 개입 루프를 즉시 중단하고 안전 모드로 전환합니다.

## 5. Grokemon Supervisor 목표 시스템

Supervisor는 에이전트가 막힘 상태에 빠졌을 때 행동의 우선순위를 재조정하여 루프를 깨뜨릴 수 있도록 목표 시스템을 관리합니다. 목표의 우선순위는 다음과 같이 정의됩니다.

1. **완료 (Hall of Fame)**: 명예의 전당 등록 및 게임 클리어.
2. **상태 관측**: 전체 상태가 미사용 상태로 방치되는 것을 방지.
3. **배틀 손실 복구**: 파티 전멸 시 포켓몬 센터에서 복구 및 재정비.
4. **현재 배틀 승리**: 진행 중인 전투에서 승리하여 화면 락 해제.
5. **이름 입력 완료**: 게임 시작 시 캐릭터 및 라이벌 이름 입력 창 해결.
6. **대화 해결**: NPC와의 대화창 루프를 A/B 버튼 입력으로 탈출.
7. **루프 복구 (막힘 감지)**: StuckDetector가 감지한 막힘 상태 해결을 위한 복구 행동 실행.
8. **스토리 목표**: 스타터 포켓몬 선택, 오키드 박사의 소포 배달, 체육관 배지 획득 등 핵심 이벤트 진행.
9. **맵 탐색 확장**: 새로운 타일 및 맵 영역 개척.
