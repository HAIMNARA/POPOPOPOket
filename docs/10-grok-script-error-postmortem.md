# 10 · 포스트모템 — "스크립트 추가 오류" 반복 & 그록 모델 효과

## 사고 개요

| 항목 | 내용 |
|------|------|
| **발생 시점** | 에이전트(Grok 모델)에게 "알아서 게임 굴려라" 지시 후 10분간 자리 비움 |
| **증상** | 돌아와 보니 게임 종료됨. "스크립트를 추가해야 한다"는 오류만 반복 노출 |
| **영향** | 자동 플레이 완전 중단. 에이전트가 게임 진행 대신 오류 메시지만 반복 |
| **근본 원인** | Lua 소켓 서버가 mGBA에 로드되지 않은 상태에서 하네스 실행 |

---

## 1. 근본 원인 분석

### 연결 체인 구조

```
에이전트 (Grok/Claude)
  → TypeScript 하네스 (HTTP 클라이언트)
    → mGBA-http.exe (HTTP:5000 → TCP:8888 브리지)
      → mGBASocketServer.lua (mGBA 내부 소켓 서버)
        → mGBA 에뮬레이터 API
```

### 실패 지점

```
에이전트 → 하네스 → mGBA-http.exe(:5000) → TCP(:8888) → ❌ Lua 미로드
                                                              ↑
                                                     이 지점에서 끊김
```

**mGBA 0.10.x에는 `--script` CLI 옵션이 없다.**
Lua 소켓 서버는 반드시 **mGBA GUI에서 수동으로 한 번** 로드해야 한다:
`Tools → Scripting → File → Load script → mGBASocketServer.lua`

### 오류 발생 시퀀스

1. 에이전트가 `start-harness.ps1` 또는 직접 mGBA + mGBA-http를 실행
2. mGBA는 정상 실행되고 ROM도 로드됨 (창 제목: `mGBA - POKEMON RED - 0.10.5`)
3. mGBA-http도 실행되어 포트 5000은 열림
4. **그러나 Lua 소켓 서버가 로드되지 않아 포트 8888이 닫혀 있음**
5. 하네스가 HTTP 요청 → mGBA-http가 TCP:8888 연결 실패 → **HTTP 500 반환**
6. 에이전트가 500 에러를 받고 "스크립트를 로드해야 한다"고 판단
7. 에이전트가 `--script` CLI 옵션으로 Lua 로드를 시도 → **`unknown option -- script` 에러로 mGBA 즉시 종료**
8. mGBA 프로세스 사망 → 게임 종료
9. 에이전트가 mGBA 재실행 → 다시 Lua 미로드 → 같은 실패 무한 반복

### 다른 사용자들은 왜 성공했나

다른 참가자들은 **수동으로 Lua를 한 번 로드한 후** 하네스를 실행했다.
`start-harness.ps1`에 "ACTION REQUIRED: GUI로 Lua를 로드하세요"라는 안내가 있지만,
에이전트에게 위임했을 때 에이전트는 이 수동 단계를 **건너뛰거나 CLI로 대체하려고 시도**했다.

## 2. 직접 검증 결과 (2026-06-09)

실제 환경에서 단계별로 검증:

| 단계 | 상태 | 결과 |
|------|------|------|
| mGBA 실행 + ROM 로드 | `mGBA - POKEMON RED - 0.10.5` | 정상 |

| mGBA-http 재시작 후 HTTP 요청 | **200 OK — `POKEMON RED`** | 전체 체인 정상 |

## 3. 해결책

### 3-1. 에이전트 지시 시 반드시 포함할 전제 조건

에이전트에게 포켓몬 레드 자동 플레이를 지시할 때, 다음을 **시스템 프롬프트 또는 초기 지시에 명시**:

```
[필수 전제 조건]
1. mGBA가 ROM과 함께 실행 중이어야 한다
2. Lua 소켓 서버가 GUI에서 한 번 로드되어 있어야 한다 (포트 8888 리슨)
3. mGBA-http가 실행 중이어야 한다 (포트 5000 응답)

[금지 사항]
- mGBA에 --script 옵션을 사용하지 마라 (존재하지 않는 옵션, mGBA 즉시 종료됨)
- Lua 스크립트를 두 번 로드하지 마라 (크래시)
- dofile()로 Lua를 로드하지 마라 (크래시)

[진단]
- HTTP 500 에러 → Lua가 로드되지 않음 → 사람이 GUI로 로드해야 함
- ECONNREFUSED → mGBA-http가 실행되지 않음
```

### 3-2. 원클릭 런처 스크립트

`oneshot2.ps1`을 기반으로, **mGBA 실행 → Lua GUI 자동 로드 → mGBA-http 실행 → 체인 검증**까지
하나의 스크립트로 자동화. 에이전트가 이 스크립트를 실행하면 수동 개입 없이 전체 체인이 구성된다.

상세: [harness/launch-pokemon.ps1](../harness/launch-pokemon.ps1) (POPOPOPOket 레포에 포함)

### 3-3. 사전 검증 가드

하네스 시작 전에 포트 8888 리슨 여부를 확인하고, 닫혀 있으면:
- Lua GUI 자동 로드를 시도 (`oneshot2.ps1` 로직)
- 실패 시 "사람이 GUI로 Lua를 로드해야 합니다" 메시지와 함께 대기

## 4. 그록(Grok) 모델 효과

### 왜 그록이 가장 효과적이었나

2026-06-06 회의 및 실제 실행 결과, **Grok 모델(grok-4.3)** 이 포켓몬 레드 자동 플레이에서
가장 효과적인 성능을 보였다:

| 모델 | 장점 | 단점 |
|------|------|------|
| **Grok 4.3** | 비전 인식 정확도 높음, 게임 컨텍스트 이해 우수, 토큰 비용 효율적 | 프록시 필요 |
| GPT-5.4 | 저비용 운용 가능 | 맵 탈출 구간에서 매핑 이슈 |
| Claude | 코딩/하네스 설계에 강함 | 비전 기반 게임 조작에서 상대적 약세 |

### 실제 하네스 환경 설정 (.env)

```
AI_BASE_URL=https://proxy.inonono.com/v1
AI_MODEL=grok-4.3
```

- 하네스는 `@ai-sdk/openai-compatible` 패키지를 사용하여 OpenAI 호환 API로 연결
- Grok 모델을 프록시 경유로 사용
- 비전(스크린샷) + RAM 상태를 동시에 입력받아 판단

### Viridian City 도달 성과

그록 모델 기반 하네스가 **침실(2F) → Viridian City** 까지 약 7시간 만에 자율 도달한 사례가
2026-06-06 이벤트에서 보고됨. 이는 다른 모델 대비 가장 빠른 기록 중 하나.

## 5. 재발 방지 체크리스트

에이전트에게 포켓몬 자동 플레이를 맡기기 전:

- [ ] mGBA가 ROM과 함께 실행 중인가?
- [ ] Lua 소켓 서버가 로드되어 포트 8888이 리슨 중인가?
- [ ] mGBA-http가 실행되어 포트 5000이 응답하는가?
- [ ] `http://127.0.0.1:5000/core/getgametitle` 이 `POKEMON RED`를 반환하는가?
- [ ] 에이전트 지시에 "스크립트 CLI 로드 금지" 명시했는가?
- [ ] `.env` 파일에 AI_MODEL과 AI_API_KEY가 설정되어 있는가?

위 체크리스트를 모두 통과한 후에만 `pnpm dev`로 하네스를 실행할 것.
